In selecting a technology in case my web app can't be accessed, I would choose local/session storage since this is supported by almost all browsers. It is the most widely used technology, so this tells me that it's a proven, consistent method.

In order to use local/session storage, I would use the JavaScript code needed to activate the storage function and use the appropriate methods and objects. Most likely, because local storage runs in the client and the JavaScript components can access features in the client, this would be the first option before using session storage, where data is stored dynamically and is only available in a particular browser session. In other words, session storage is not persistent and therefore, not the best option for caching while local storage is persistent.
